## Ticket template

Mỗi ticket file bao gồm:

```yaml
---
id: LIQ-XXX
title: ...
phase: 1 | 2 | 3
priority: P0 | P1 | P2
estimate: Xd
status: Backlog | In Progress | In Review | Done | Blocked
depends_on: [LIQ-YYY]
blocks: [LIQ-ZZZ]
tags: [backend, frontend, db, ai, ...]
---
```

Nội dung: **Problem → User story → Acceptance criteria → Technical approach → Data model → API → UI notes → Testing → Out of scope**.

## Naming / workflow

- Branch: `feature/LIQ-XXX-short-slug`
- PR title: `feat(scope): LIQ-XXX — short description`
- Commit message reference: `LIQ-XXX`
- Update ticket status + this README khi merge
